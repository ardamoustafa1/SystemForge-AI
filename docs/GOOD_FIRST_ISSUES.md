# Good First Issues

Use these as starter tasks for contributors. They are intentionally scoped so a new contributor can make a useful PR without understanding the entire system.

## Documentation

- Add screenshots for a specific feature page.
- Improve a runbook with exact reproduction commands.
- Add a new ADR for a small architectural decision.
- Expand one showcase example with an export artifact.

## Frontend

- Add accessible labels to one dashboard control.
- Add empty/loading/error state coverage for one page.
- Add a Vitest test for a schema or utility.
- Improve mobile spacing in the landing navigation.

## Backend

- Add a focused unit test for one service function.
- Add validation coverage for one API schema.
- Improve an error message with an actionable response.
- Add a small health/readiness assertion to tests.

## Ops

- Add a k6 scenario for a read-heavy dashboard flow.
- Add a Helm values example for staging.
- Add a dashboard panel to `ops/grafana`.
- Add a runbook section for one alert.

When opening a PR, include:

- What changed.
- Why it changed.
- How you tested it.
